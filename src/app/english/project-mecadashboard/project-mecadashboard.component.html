Mecadashboard

Project Description

This project is a monitoring dashboard for the Meca500 robot from Mecademic. It uses their Python driver with Flask to communicate with the robots and the front-end is built with React. You can add robots to the list and you can then monitor the status of these robots. This can be useful in a situation where there are many robots on the same network and they have to be monitored.

Motivation

This project was created in part to practice React, but also to show our customers that you can build a web app to interact with mutiple robots. This can also be a good tool to monitor a lot of robots during trade shows and quickly see if all the demos are running properly.

I also built this to better understand how to create a micro-service using Flask. I have started another project to expand the backend to support more commands for a robot.

How it works

Once the backend server is booted and the React app is running, you can connect using your web browser. You get to the main page of the application where there's a link to the documentation and one to the support.

Then you can click on the + button to add a robot to the list. Enter the IP address of the robot and give it a name. If the robot is available, the backend will make a connection but if not an error is returned.

You can add many robots as long as you can connect to them. If you try to add a robot that is already on the list, an error will be shown. Note that this app only works if you have robots available.

Tech Used

The frontend was built using React. Flask is used to manage the HTTP request on the server side. Python is used for the communication with the actual robots.
